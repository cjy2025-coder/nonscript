# NonScript 语法规范补充手册（完整整合版）

# 注释系统

NonScript 支持标准单行注释与块级多行注释，语法贴近主流编程语言，可读性强，解析器原生支持，不影响 AST 生成与代码编译。

```Plain Text
var a=1;# 单行注释
 /* var a=1 */ 多行块注释

```

# 基础类型对照表

NonScript 为**静态强类型语言**，所有基础类型都会一对一无损映射到标准 C\+\+ 原生类型，保证编译后零开销、内存布局对齐、运行性能等同于原生 C\+\+。

|NonScript 类型|底层等价 C\+\+ 原生类型|
|---|---|
|i8|signed char|
|i16|short|
|i32|int|
|i64|long long|
|f32|float|
|f64|double|
|bool|bool|
|string|std::string|
|void|void|
|object|类对象指针|
|func|函数指针|
|array|std::variant\&lt;\&gt; 泛型容器|

# 库导入机制

支持模块化文件导入，拆分工程代码，方便大型项目分工开发、复用公共工具库代码，提升项目可维护性。

当前阶段能力说明：解释器模式已完整支持跨文件导入解析，可直接加载外部 `\.ns` 库文件并合并执行；C\+\+ 编译模式正在优化全局符号合并、跨文件 AST 拼接逻辑，后续版本将全量支持编译期批量导入链接。

```Plain Text
import "lib.ns"

```

# 变量 / 常量声明规范

兼顾动态推导便捷性与静态类型安全性，提供多档位声明写法，适配快速开发与严谨工程开发两种场景，规避隐式类型精度丢失问题。

```Plain Text
var a; # 纯声明变量，不指定类型、不初始化，类型暂不可知
 var a:i64; # 显式指定类型，仅声明不赋值
 var a = 10; # 自动类型推导，根据右侧值反向识别类型
 var a:i64 = 10; # 强推荐严谨写法，固定类型，杜绝推导精度缺失
 const a:i64 = 10; # 常量定义，必须强制初始化，运行期不可修改

```

# 函数系统完整规范

## 函数声明

### 显式具名函数

支持完整返回值标注与返回值自动推导两种模式，工程开发优先使用显式返回值写法，提升代码可读性与安全性。

```Plain Text
# 标准严谨写法：显式标注返回值类型，推荐所有工程场景使用
  func f(i64 n) -> void{}
  # 简写写法：缺省返回值，编译器自动推导返回类型，存在轻微精度丢失风险
  func f(i64 n){}

```

### 匿名函数

无函数名，直接赋值给函数类型变量，适用于回调逻辑、临时函数场景，语法简洁灵活。

```Plain Text
# 无函数名，直接绑定变量，可直接当作参数传递、回调使用
  var myFunc=func()->void{}

```

### 外部对接函数（extern 跨语言调用）

专门用于对接底层原生 C\+\+ 全局函数，仅声明函数签名、无函数体，编译期直接符号绑定，无缝打通 NonScript 与 C\+\+ 底层能力，拓展系统调用、硬件操作、第三方库能力。

```Plain Text
/*
  * 通过 extern 关键字声明外部链接函数
  * 无函数体，必须完整标注返回值与参数类型
  * 用于直接绑定底层原生 C++ 函数
  */
  extern f(i64 n)->void

```

配套底层 C\+\+ 对接实现示例，一一对应即可编译通过：

```Plain Text
void f(long long n){
      // 原生 C++ 底层逻辑实现
  }

```

## 函数调用语法

兼容常规主流编程语言调用习惯，直接传参即可，支持常量、变量、表达式、函数嵌套传参。

```Plain Text
f(0)

```

# 面向对象：类系统全规范

## 类声明与继承

支持多父类公有继承，使用 extends 关键字快速声明父类，默认全部为公有继承模式，简化面向对象层级设计。

```Plain Text
# 单类多继承，自动继承所有合法成员与成员方法
  class A extends B,C{

  }

```

## 类成员访问权限控制

严格面向对象封装特性，支持三层访问修饰符，未手动标注权限时，默认按 private 私有成员处理，保障数据封装安全。

```Plain Text
class A extends B,C{
     private var a:i64 = 0        # 私有成员：仅本类内部访问
     protected const b:string = "hello"
     public func f()->void{}     # 公有成员：外部可直接调用访问
  }
```

## 类对象实例化

通过 new 关键字动态创建堆区对象，自动绑定类内存布局，当前已支持无参实例化，后续迭代补充带参构造函数重载能力。

```Plain Text
# 两种等价实例化写法，自动分配内存、自动管理对象生命周期
  var a1=new A()
  var a2:A = new A

```

## 类成员方法与成员变量访问

使用点语法直接访问公有成员与方法，语法直观贴合主流面向对象语言习惯。

```Plain Text
a.f()

```

# 流程控制语句

## 分支语句

if\-else 多分支语句、switch\-case 匹配分支语句，语法、语义、执行逻辑完全对齐标准 C\+\+，上手零学习成本，解析器与编译器双模式全量兼容支持。

## 循环语句

- **while 循环**：标准前置条件循环，逻辑与 C\+\+ 完全一致，稳定无 bug。

- **until 循环**：反向条件循环，自动取反 while 判断条件，简化反向循环业务写法。

- **for 循环**：兼容常规三段式循环语法，基础可用，当前存在小幅适配 BUG，后续版本迭代修复优化。
