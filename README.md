# NonScript —— 静态强类型双模式编程语言

**NonScript** 是一款基于 cpp 实现的、**静态强类型、面向对象、双模式执行**的自研小型编程语言。内置完整词法分析、语法分析、语义分析、AST 抽象语法树、解释器、cpp 转译编译器全套链路，既可**解释直接运行**，也可**编译为原生高性能 EXE**。

---

##  一、基础介绍

- **语言定位**：轻量现代自定义编程语言，兼顾易用性、可扩展性、高性能。

- **类型系统**：静态强类型，编译期类型检查，杜绝动态类型运行时报错。

- **运行模式**：双架构并存 —— 解释执行&编译执行。

- **底层实现**：纯 c++ 手写递归下降分析器，无第三方语法库、无依赖虚拟机。

- **适用场景**：教学演示、编译原理课程设计、小型脚本工具、自研语言实践项目。

---

## 二、核心亮点

### 1. 双模式运行架构

同一份源码，两种运行方式：

- **解释器模式**：直接遍历 AST 执行，无需生成 c++、无需调用编译器，秒启动、适合调试。

- **编译模式**：AST 直接高质量转译为标准 c++ 源码，调用 GCC 生成原生机器码，**运行速度接近原生 c++**。

### 2. 完整面向对象能力

- 支持 class 类定义、成员变量、成员函数

- 支持 new 实例化对象

- 编译器自动生成对象序列化 __toString__ 函数，自动打印对象信息，同时也可在类中重写 __toString__ 函数来实现自定义的序列化

### 3. 内置函数

- 控制台输入输出：print / scan

### 4. 完善类型系统

- 整型：i8 / i16 / i32 / i64

- 浮点：float / double

- 布尔、字符串、函数类型、自定义类实例对象

- 语义分析阶段严格类型校验，杜绝类型不匹配错误
---

## 三、整体技术架构

```Plain Text
NonScript 源码文本
        ↓
词法分析 Lexer（切割 Token）
        ↓
语法分析 Parser（递归下降 → 生成 AST）
        ↓
语义分析 Semantic（类型检查、作用域、函数签名校验）
        ↓
        ┌─────────────┴─────────────┐
        ↓                           ↓
      解释器                     代码生成 
直接跑AST，快速调试            输出标准C++源码
                                    ↓
                               GCC 编译链接
                                    ↓
                            原生 exe 可执行文件
    
```

---

## 四、运行效率说明

- **编译模式性能**：直接等价 c++ 运行速度，无虚拟机、无 GC、无解释层开销，性能远超 Python / JS / Java。

- **解释模式性能**：适合快速迭代开发、调试语法、验证逻辑，启动速度快。
---

## 五、目前已完整支持的功能清单

### 基础语法

- 静态类型变量定义 var 变量:类型，当类型缺省时则尝试自动推导

- 完整表达式运算：加减乘除、逻辑运算、比较运算

- 代码块、作用域管理

### 函数系统

- 全局函数、带参数、带返回值

- 函数类型识别，自动打印函数地址

- 外部函数 extern 声明对接底层 c++

### 面向对象系统

- 类定义、成员变量、成员函数

- 对象实例化 new

- 自动生成 __toString__ 序列化方法

- 成员访问

### 输入输出系统

- print 普通打印

- scan 控制台读取数字

- 自动格式化所有数据类型输出

### 文件操作系统（暂未添加）

- 类成员非法定义报错
